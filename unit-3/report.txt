UNIT 3 REPORT

1. Insertion Sort
- Time Complexity:
  Best Case: O(n)
  Worst Case: O(n²)
- Stable: Yes
- In-place: Yes

2. Merge Sort
- Time Complexity:
  Best/Worst Case: O(n log n)
- Stable: Yes
- In-place: No

3. Quick Sort
- Average Case: O(n log n)
- Worst Case: O(n²)
- Stable: No
- In-place: Yes

Observation:
- Insertion Sort works well for small datasets.
- Merge Sort gives stable performance.
- Quick Sort is fast on average but worst case occurs on sorted data.
